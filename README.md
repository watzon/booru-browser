# Booru Browser

A fast, private, one-handed client for browsing booru-style image boards — all your boards in one app. Built with Expo (React Native).

> Booru Browser is a client for third-party image boards and is not affiliated with any of them. You bring your own sources.

## Features

- **One app for every booru.** Add the boards you use and switch between them instantly. Works with Danbooru, Gelbooru, Safebooru, e621, Konachan/Moebooru, Realbooru, Kemono, and custom sites via pluggable adapters.
- **No bundled server list.** Add servers manually, import a `.booruconfig.json`, or scan a QR code.
- **Private by design.** No account, no analytics, no tracking. Servers and API keys are stored only on your device (keys in the iOS Keychain via `expo-secure-store`).
- **SFW by default.** Mature content stays locked behind an 18+ confirmation and an optional Face ID / passcode check (Settings → Content → Mature content).
- **Browse your way.** Endless, virtualized grids (`@shopify/flash-list` + `expo-image` caching) in grid, bento, or full-card layouts.
- **One-handed navigation.** A floating control puts browsing, search, and instant booru-switching under your thumb; swipe left/right to move between Browse, Favorites, and Settings.
- **Tag search** with per-source autocomplete and recent tags.
- **Local favorites + downloads** to Photos, with share-sheet support.

## Development

Expo SDK 54, React Native 0.81, new architecture, React Compiler. Managed with **pnpm**.

```sh
pnpm install
pnpm ios        # or: pnpm android
pnpm preflight  # lint + typecheck + expo-doctor (run before building)
```

A **development build is required** (not Expo Go) because of `expo-camera`, `expo-media-library`, `expo-secure-store`, and `expo-local-authentication`:

```sh
pnpm exec eas build --profile development --platform ios
```

`@/*` resolves to the repo root (see `tsconfig.json`); use it for all internal imports. `ios/` and `android/` are generated by Expo (continuous native generation) and are gitignored — edit `app.json`/config plugins, not the native projects.

## Importing a server list

The app deep-links via `boorubrowser://`. Two payload styles are supported:

```
boorubrowser://import?src=https://example.com/list.json
boorubrowser://import?data=<base64url-encoded JSON>
```

QR scanning hits the same import path. A sample list lives in [`examples/sample.booruconfig.json`](examples/sample.booruconfig.json).

## Architecture

```
sources/       Source adapter interface + per-family adapters (danbooru, gelbooru, e621, moebooru, kemono, custom)
servers/       User's server list (zustand + AsyncStorage) and auth credentials (secure-store)
gate/          NSFW age gate (secure-store) and the single rating-filter enforcement point
favorites/     Locally-saved posts
downloads/     File-system + share-sheet helpers
services/      Cross-cutting services (e.g. webview fetcher)
components/    UI primitives (post grid, viewer, server form, tag search)
hooks/         React Query hooks + zustand search-state store
app/           expo-router screens (typed routes)
```

The age gate is structural, not advisory: every `Source.search` call passes through `applyRatingFilter` (and post detail through `shouldHidePost`), so UI code cannot accidentally surface mature content while the gate is locked.
